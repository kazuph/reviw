# /reviw-plugin:ask

<command-name>ask</command-name>

ユーザーの要求を勝手に解釈せず、AskUserQuestionを使って深くインタビューします。

## このコマンドが呼ばれた理由

ユーザーは以下のような不満を感じています：
- Claudeが勝手に解釈して進めている
- 本当にやりたいことと違うことをやっている
- 質問せずに推測で動いている

## 必須アクション

**今すぐAskUserQuestionツールを使って、以下を確認してください：**

1. **ゴール確認**
   - 「最終的に何を達成したいですか？」
   - 「なぜそれが必要ですか？」

2. **スコープ確認**
   - 「今回のタスクの範囲はどこまでですか？」
   - 「やらなくていいことは何ですか？」

3. **制約確認**
   - 「技術的な制約はありますか？」
   - 「時間的な制約はありますか？」

4. **優先順位確認**
   - 「複数の要求がある場合、何が最優先ですか？」

5. **成功基準確認**
   - 「何ができたら完了ですか？」
   - 「どうやって確認しますか？」

## 禁止事項

- ❌ 「たぶんこういうことだろう」と推測して進める
- ❌ 質問せずに実装を始める
- ❌ 「〜ということでよろしいですか？」と確認形式で進める（ちゃんと質問しろ）

## 実行形式

```
AskUserQuestionツールを使って、選択肢付きで質問する
```

質問は1つずつ、明確に、選択肢を提示して行うこと。
